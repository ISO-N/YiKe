YiKe / Material 3 Mobile Prototype

忆刻手机 App UI/UX 原型

这一版围绕“打开就能开始复习”来设计，采用 Material 3 常见的色彩语义、卡片层级、 底部导航和 FAB 语言，同时保留离线学习工具应有的克制感。

主色 / Primary

用于开始复习、关键确认按钮和流程进度，保持任务导向感。

容器色 / Primary Container

承接今日任务、轻状态提示与顶部概览，减少大面积纯色压迫。

强调色 / Tertiary

用于 FAB、备份风险和轻提醒，让高关注动作更容易被看到。

表面层 / Surface

大面积背景保持浅色高对比，更适合长时间阅读题面和答案。

信息架构

首页、卡组、设置三段式底部导航承接日常入口；复习流作为独立沉浸式页面； 卡组详情和问题编辑延续 Material 列表与表单逻辑，减少学习成本。

01 / Home

首页

任务总览、开始复习、近期卡组入口

首页需要在几秒内说明“今天要做什么”，所以保留大卡片概览、主操作按钮和近期卡组， 不把内容管理信息挤进首屏。

单独打开 看下一页

02 / Review

复习页

问题聚焦、显示答案、四档评分

复习页本质上是沉浸式任务页，因此让题面和评分区拥有最大的纵向空间， 减少用户把注意力浪费在导航和装饰上。

单独打开 看下一页

03 / Decks

卡组列表

主题聚合、新建卡组、今日到期信息

卡组页承担的是“整理学习主题”的职责，所以列表项中同时显示规模和今日到期量， 让管理动作与复习优先级能够一起被看见。

单独打开 看下一页

04 / Cards

卡片列表

章节级内容管理、进入问题编辑

卡片层相当于主题下的章节目录，保留相同的列表视觉语言， 可以让用户把认知资源留给内容本身，而不是新的页面规则。

单独打开 看下一页

05 / Editor

问题编辑

卡片信息 + 多问题编辑，贴近移动端表单习惯

编辑页优先保证输入效率，因此采用长页面单列结构，而不是在移动端硬塞双栏， 让用户能连续录入多个问题而不频繁切换焦点。

单独打开 看下一页

06 / Settings

设置

提醒、版本信息、备份恢复入口

设置页承接的是低频但重要的全局能力，因此内容分组更克制， 重点放在提醒与数据安全，而不是堆叠大量系统项。

单独打开 看下一页

07 / Backup

备份与恢复

高风险操作清晰分层，减少误触

这类页面最怕“看起来轻松，实际上危险”，因此把风险说明抬到页面顶部， 并给导出与恢复两种动作明显区分的按钮层级。

单独打开 回到顶部
