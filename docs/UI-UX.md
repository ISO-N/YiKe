- 作者：YiKe 团队

---

## 1. 设计目标与原则

### 1.1 设计目标

- **零门槛**：输入即规划，用户不必理解记忆曲线理论。
- **专注高效**：关键路径短、反馈明确、尽量减少打断。
- **跨平台一致**：移动端与 Windows 桌面端保持一致的信息架构与视觉语言。
- **可访问**：对比度、触摸目标、键盘与读屏适配可用。

### 1.2 交互原则（落地口径）

- 任何“重要操作”（导入、删除、清空、同步、生成模拟数据）都必须具备：确认/进度/结果反馈。
- 空状态要“可行动”：告诉用户下一步做什么，而不是只提示“暂无数据”。
- 状态表达一致：待复习/已完成/已跳过/逾期等在所有页面保持同一套颜色与标签体系。

## 2. 设计体系（Design Tokens）

### 2.1 视觉风格

- 风格：Glassmorphism（毛玻璃）+ 渐变背景（轻层次）。
- 使用场景：卡片、弹窗、底部导航、进度展示。

### 2.2 配色（摘要）

| Success | `#22C55E` | 完成状态 |
| Warning | `#F59E0B` | 逾期/警告 |

## 3. 信息架构与导航

### 3.1 底部导航（5 Tab）

| Tab | 路由 | 说明 |
|---|---|---|
| 今日 | `/home` | 今日任务与快速操作 |
| 日历 | `/calendar` | 月视图与当天列表 |
| 统计 | `/statistics` | 完成率与分布 |
| 帮助 | `/help` | 学习指南（Markdown 渲染） |
| 设置 | `/settings` | 提醒/主题/导出/同步/关于 |

### 3.2 Windows 桌面端导航补充

- 保持同一 IA，但允许更高信息密度（例如列表双列、侧栏/顶部工具条）。
- 快捷键：常用操作可映射（如新建/刷新/搜索聚焦）。

## 4. 页面规范（关键路径）

### 4.1 首页（今日任务）

- 结构：日期/搜索（v3.1）/进度（v3.1）/任务列表/浮动按钮（新建）。
- 任务卡片：标题 + 标签 + 第 N 次复习 + 状态控件（勾选/跳过）。
- 手势：左右滑快速完成/跳过（移动端）；桌面端用按钮/右键菜单替代滑动。

### 4.2 录入页（含 v2.1 增强）

- 输入：标题/备注/标签/学习日期。
- 批量导入入口：导入文件 → 预览/编辑 → 确认保存。
- 模板：选择模板后填充字段；占位符需可预期（预览替换结果）。
- 复习预览面板：显示将生成的复习节点；配置入口清晰。

### 4.3 日历页（含 v3.1 筛选）

- 月历 + 当日任务列表。
- 日期标记：用颜色/圆点表达“有任务/逾期/完成”等（保持全局一致）。
- 筛选栏（v3.1）：全部/待复习/已完成/已跳过；同时展示每类数量。

### 4.4 统计页

- 关键指标卡：连续天数、周/月完成率。
- 分布：标签/主题分布（饼图或等价图表）。
- 空状态：提示用户先录入内容或完成任务。

### 4.5 导出页

- 选择导出对象（学习内容/复习任务）与格式（JSON/CSV）。
- 导出过程：加载态 → 成功提示（可分享）/失败提示（可重试）。
- 默认排除 Mock 数据（v3.1 设计口径）。

### 4.6 设置页

- 提醒：提醒时间/免打扰。
- 外观：主题模式（跟随系统/浅色/深色）。
- 数据：导出、（可选）清理 Mock 数据。
- 同步：自动同步开关、设备列表、手动同步、主机设置（如果有）。
- Debug（仅 Debug）：模拟数据生成入口（v3.1）。

### 4.7 帮助页（学习指南，F10）

- 标题 + 副标题 + 可滚动正文；提供目录跳转（锚点/列表）。
- Markdown 样式：标题层级、列表、表格、引用、代码块在深浅模式下可读。

### 4.8 同步状态（F12）

- 状态定义：未连接/发现中/配对中/已连接/同步中/失败。
- 展示位置：设置页同步区块 + 顶部/列表轻提示 +（桌面端）托盘菜单可显示状态。
- 交互反馈：配对成功/失败、同步成功/失败、断开连接都必须有明确提示。

### 4.9 Debug 模拟数据生成器（F13，v3.1）

- 入口：设置页（仅 Debug 可见）。
- 表单：学习内容数量、复习任务数量、日期范围、模板类型；禁用态与范围校验明确。
- 结果：生成成功提示 + 可一键清理（只清理本次生成或按 isMockData 规则清理）。

## 5. 组件规范（摘要）

### 5.1 按钮

| 类型 | 用途 |
|---|---|
| Primary | 主要行动（保存、导出、同步、生成） |
| Secondary | 次要行动（取消、返回） |
| Ghost | 文字按钮/轻操作 |
| Danger | 删除/清空/断开等风险操作 |

### 5.2 状态标签与颜色

- 待复习：Primary/信息色
- 已完成：Success
- 已跳过：Text Secondary/灰
- 逾期：Warning（或 Error，按产品口径）

### 5.3 进度环（v3.1）

- 未开始：灰色
- 进行中：Primary（Teal）
- 已完成：Success（Green）
- 文案：`已完成/总数`，点击可展开详情。

---

## 6. 动画与反馈（摘要）

- 页面切换：300ms ease-in-out。
- 主题切换：背景渐变 300ms；卡片轻缩放 100ms。
- 关键操作（导入/导出/同步/生成）：必须有加载态与结束态提示。

---

## 7. 响应式与桌面端规范

### 7.1 建议断点

- 移动端：< 600px
- 平板：600px - 1024px
- 桌面端：≥ 1024px（可使用双列/侧栏）

### 7.2 Windows 窗口规范（建议值）

- 默认：1200×800；最小：800×600。
- 托盘：最小化到托盘；托盘菜单展示同步状态与快捷入口。

---

## 8. 无障碍（摘要）

- 对比度：正文 ≥ 4.5:1；大字 ≥ 3:1。
- 触摸目标：≥ 44×44；列表项高度 ≥ 48。
- 读屏：按钮/输入框有语义标签；状态变化可被朗读。
- 键盘：桌面端核心操作可聚焦、可触发。

---

## 9. 验收检查清单（UI/UX）

- 深浅主题：所有页面、图表、日历组件在两种主题下可读。
- 帮助页：Markdown 样式完整、可滚动、目录可跳转。
- 同步：配对/同步/失败/断开均有可理解反馈；状态展示一致。
- v3.1：搜索/筛选/进度展示不遮挡核心任务，且与数据口径一致。
