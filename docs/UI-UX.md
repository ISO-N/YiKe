# 忆刻（YiKe）— UI/UX 设计（汇总版）

## 文档信息

- 用途：统一沉淀设计体系、交互规则与验收检查点，避免多版本 UI 文档重复维护。
- 作者：YiKe 团队
- 创建日期：2025-02-25
- 最后更新：2026-03-05

---

## 1. 设计目标与原则

### 1.1 设计目标

- **零门槛**：输入即规划，用户不必理解记忆曲线理论。
- **专注高效**：关键路径短、反馈明确、尽量减少打断。
- **跨平台一致**：移动端与 Windows 桌面端保持一致的信息架构与视觉语言。
- **可访问**：对比度、触摸目标、键盘与读屏适配可用。

### 1.2 交互原则（落地口径）

- 任何“重要操作”（导入、删除、清空、同步、生成模拟数据）都必须具备：确认/进度/结果反馈。
- 空状态要“可行动”：告诉用户下一步做什么，而不是只提示“暂无数据”。
- 状态表达一致：待复习/已完成/已跳过/逾期等在所有页面保持同一套颜色与标签体系。

### 2.1 视觉风格

- 风格：Glassmorphism（毛玻璃）+ 渐变背景（轻层次）。

## 3. 信息架构与导航

### 3.1 底部导航（4 Tab）

| Tab | 路由 | 说明 |
|---|---|---|
| 今日 | `/home` | 今日任务与快速操作 |
| 日历 | `/calendar` | 月视图与当天列表 |
| 统计 | `/statistics` | 完成率与分布 |
| 设置 | `/settings` | 提醒/主题/导出/同步/关于 |

**全量任务入口（已收敛）**

- 全量任务已收敛到首页内切换：`/home?tab=all`（页面内“今日 / 全部”）。
- 旧路由兼容：`/tasks` 通过 redirect 跳转到 `/home?tab=all`（避免深链断裂）。

### 3.1.1 帮助页（独立路由）

- 路由：`/help`
- 形态：独立全屏页面（不在底部导航内，不显示底部导航栏）
- 入口：设置页「帮助与学习指南」；桌面端支持快捷键（Ctrl+H）
- 兼容路由：`/settings/help` redirect 到 `/help`

### 3.2 Windows 桌面端导航补充

- 保持同一 IA，但允许更高信息密度（例如列表双列、侧栏/顶部工具条）。
- 快捷键：常用操作可映射（如新建/刷新/搜索聚焦）。

### 3.3 IA 精简重构（规划口径）

> 本节为规划口径：用于指导“入口聚合/路由兼容/页面内切换”的逐步收敛，不要求一次性大改。

- 目标方向：4 Tab → 3 Tab（今日/日历/设置）；统计能力以 Sheet/详情形式整合到日历页。
- 路由兼容：保持旧路由可访问，通过 redirect 或 query 参数触发对应视图（例如统计自动展开）。

## 4. 页面规范（关键路径）

### 4.1 首页（今日/全部任务）

- 结构：搜索/更多 → “今日/全部”切换 → 进度 → 筛选条 → 任务列表 → Shell FAB（新建）。
- 任务卡片信息：标题 + 标签 + 第 N 次复习 + 描述/子任务摘要 + 状态控件。
- 任务卡片交互：
  - 点击：展开/收起详情（展示描述、子任务、完成/跳过/撤销/查看详情等）。
  - 长按（移动端）/右键（桌面端）：上下文菜单（完成/跳过/撤销/查看详情，按状态变化）。
  - 左滑（移动端）：快捷完成/跳过（仅 pending；选择模式禁用；done/skipped 禁用）。
- 空状态分层：
  - 冷启动（全库内容=0）：引导“添加学习内容 / 创建专题”。
  - 常规空态（全库内容>0 且今日无待复习）：提示“今日任务已完成”并提供添加入口。
- 失败回滚口径：所有状态变更操作采用“操作先行”；失败不播放移除动效，SnackBar 提示可重试。
- 减少动效：系统开启“减少动态效果”时，完成动效/进度动画/触觉反馈需降级或禁用。

### 4.2 录入页（含 v2.1 增强）

- 输入：标题/描述/子任务/标签/学习日期（`note` 仅作为历史兼容字段，不作为主要编辑入口）。
- 批量导入入口：导入文件 → 预览/编辑 → 确认保存。
- 导入引导：导入预览页提供“格式说明 + 下载模板（TXT/CSV/Markdown）”，避免用户无从下手。
- 模板：选择模板后填充字段；占位符需可预期（预览替换结果）。
- 复习预览面板：显示将生成的复习节点；配置入口清晰。

### 4.2.1 任务详情 Sheet（学习内容视角）

- 路由：`/tasks/detail/:learningItemId`（以 learningItemId 为参数，便于展示“学习内容信息 + 全部轮次计划”）。
- 操作区（按状态/是否停用变化）：
  - 编辑：修改描述/子任务/标签（影响同一学习内容的所有轮次任务展示）。
  - 调整计划：仅允许调整 pending 轮次；日期选择需满足相邻轮次约束。
  - 增加/减少轮次：最大 10、最小 1；减少轮次需二次确认并提示对历史统计影响。
  - 停用学习内容：软删除入口；停用后详情页只读并禁用所有操作按钮。

### 4.2.2 主题管理页（Topics）

- 路由：`/topics`（不在底部导航内，作为独立管理页面）。
- 首次引导：首次进入显示“主题用途 + 使用步骤”说明卡片/弹窗；支持“我知道了/不再提示”并记忆。
- 目标：降低“主题是什么、怎么用”的理解成本，避免用户因为不理解而放弃使用。

### 4.3 日历页（含 v3.1 筛选）

- 月历 + 当日任务列表。
- 日期标记：用颜色/圆点表达“有任务/逾期/完成”等（保持全局一致）。
- 筛选栏（v3.1）：全部/待复习/已完成/已跳过；同时展示每类数量。
- 统计整合（规划）：日历页顶部可展示紧凑统计栏（连续打卡/本周完成率），点击展开统计详情 Sheet。

### 4.4 统计页

- 关键指标卡：连续天数、周/月完成率。
- 分布：标签/主题分布（饼图或等价图表）。
- 空状态：提示用户先录入内容或完成任务。
- 统计增强（规划）：趋势图表、热力图、目标设定、对比分析等能力逐步补齐。

### 4.5 导出页

- 选择导出对象（学习内容/复习任务）与格式（JSON/CSV）。
- 导出过程：加载态 → 成功提示（可分享）/失败提示（可重试）。
- 默认排除 Mock 数据（v3.1 设计口径）。

### 4.5.1 备份与恢复页

- 入口：设置页 → 备份与恢复（数据管理区块）。
- 页面结构：快捷按钮（导出备份/导入备份）+ 备份历史列表 +（可选）导入前快照入口。
- 导出交互：默认导出到应用私有目录并写入历史；成功后提供“分享/另存为”。
- 导入交互：选择文件 → 校验（checksum）→ 预览条数/版本 → 选择策略（合并/覆盖）→ 执行导入 → 重建通知/刷新。
- 安全提示：覆盖导入必须二次确认；覆盖前自动创建快照；失败可回滚。
- 可取消：导入/导出过程需提供取消与中断清理策略（不留半文件/半数据）。

### 4.6 设置页

- 提醒：提醒时间/免打扰。
- 外观：主题模式（跟随系统/浅色/深色）。
- 数据：导出、备份/恢复、同步等入口聚合到“数据管理”区块；（可选）清理 Mock 数据。
- 同步：自动同步开关、设备列表、手动同步、主机设置（如果有）。
- Debug（仅 Debug）：模拟数据生成入口（v3.1）。
- 体验偏好（规划/增量）：触觉反馈开关、撤销 Snackbar 开关、降低模糊/降低动效等“按设备本地”偏好。

### 4.7 帮助页（学习指南，F10）

- 标题 + 副标题 + 可滚动正文；提供目录跳转（锚点/列表）。
- Markdown 样式：标题层级、列表、表格、引用、代码块在深浅模式下可读。

### 4.8 同步状态（F12）

- 状态定义：未连接/发现中/配对中/已连接/同步中/失败。
- 展示位置：设置页同步区块 + 顶部/列表轻提示 +（桌面端）托盘菜单可显示状态。
- 交互反馈：配对成功/失败、同步成功/失败、断开连接都必须有明确提示。

### 4.9 Debug 模拟数据生成器（F13，v3.1）

- 入口：设置页（仅 Debug 可见）。
- 表单：学习内容数量、复习任务数量、日期范围、模板类型；禁用态与范围校验明确。
- 结果：生成成功提示 + 可一键清理（只清理本次生成或按 isMockData 规则清理）。

## 5. 组件规范（摘要）

### 5.1 按钮

| 类型 | 用途 |
|---|---|
| Primary | 主要行动（保存、导出、同步、生成） |
| Secondary | 次要行动（取消、返回） |
| Ghost | 文字按钮/轻操作 |
| Danger | 删除/清空/断开等风险操作 |

### 5.2 状态标签与颜色

- 待复习：Primary/信息色
- 已完成：Success
- 已跳过：Text Secondary/灰
- 逾期：Warning（或 Error，按产品口径）

### 5.3 进度环（v3.1）

- 未开始：灰色
- 进行中：Primary（Teal）
- 已完成：Success（Green）
- 文案：`已完成/总数`，点击可展开详情。

## 6. 动画与反馈（摘要）

- 页面切换：300ms ease-in-out。
- 主题切换：背景渐变 300ms；卡片轻缩放 100ms。
- 关键操作（导入/导出/同步/备份/恢复/生成）：必须有加载态与结束态提示。
- 任务完成动效（体验增强）：
  - 按钮/菜单触发：缩放 + 淡出（≤300ms）。
  - 左滑 Dismissible：保持原生滑出效果，避免叠加动效。
  - 失败不播放移除动效，保持任务可见并提示错误。
- 进度环动效（体验增强）：数值变化时平滑过渡，环形与数字同步变化。
- 无障碍降级：系统开启“减少动态效果”时，上述动效时长设为 0 或直接禁用；触觉反馈随之降级。

## 7. 响应式与桌面端规范

### 7.1 建议断点

- 移动端：< 600px
- 平板：600px - 1024px
- 桌面端：≥ 1024px（可使用双列/侧栏）

### 7.2 Windows 窗口规范（建议值）

- 默认：1200×800；最小：800×600。
- 托盘：最小化到托盘；托盘菜单展示同步状态与快捷入口。

---

## 8. 无障碍（摘要）

- 对比度：正文 ≥ 4.5:1；大字 ≥ 3:1。
- 触摸目标：≥ 44×44；列表项高度 ≥ 48。
- 读屏：按钮/输入框有语义标签；状态变化可被朗读。
- 键盘：桌面端核心操作可聚焦、可触发。

---

## 9. 验收检查清单（UI/UX）

- 深浅主题：所有页面、图表、日历组件在两种主题下可读。
- 帮助页：Markdown 样式完整、可滚动、目录可跳转。
- 同步：配对/同步/失败/断开均有可理解反馈；状态展示一致。
- v3.1：搜索/筛选/进度展示不遮挡核心任务，且与数据口径一致。
- 备份/恢复：校验/预览/合并/覆盖/快照/取消路径完整，且覆盖导入有强提示。
- 任务操作：选择模式与长按/右键/左滑互斥规则正确；done/skipped 状态菜单项正确；撤销有副作用提示。
- 性能降级：大列表滚动不卡顿；必要时可通过“降低模糊/降低动效”开关获得明显改善。
